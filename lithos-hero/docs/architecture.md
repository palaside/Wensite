# รายงานการออกแบบโครงสร้างสถาปัตยกรรมดิจิทัลสำหรับแอปพลิเคชันผู้ตรวจการณ์หน้า (ผตน.)

## 1. วัตถุประสงค์และภาพรวมของระบบ (System Purpose & Overview)
แอปพลิเคชันนี้ถูกออกแบบเพื่อเปลี่ยนผ่านกระบวนงานตามหลักนิยม (Doctrine) ของเหล่าทหารปืนใหญ่ให้เข้าสู่ระบบดิจิทัล โดยมี ผตน. เป็นศูนย์กลางในฐานะ "ดวงตา" ของชุดหลักยิง ระบบต้องรองรับภารกิจหลัก 4 ประการตามระเบียบปฏิบัติ ดังนี้:
- **การค้นหาเป้าหมาย (Target Acquisition):** พิสูจน์ทราบและกำหนดพิกัดเป้าหมายให้มีความแม่นยำสูงในเวลาที่สั้นที่สุด
- **การปรับการยิง (Adjustment of Fires):** คำนวณค่าแก้ทางเทคนิคเพื่อนำตำบลระเบิดเข้าสู่เป้าหมายอย่างมีประสิทธิภาพ
- **การตรวจผลการยิง (Surveillance of Fires):** บันทึกและรายงานความเสียหายที่เกิดขึ้นแก่เป้าหมาย รวมถึงลักษณะการระเบิด
- **การเฝ้าตรวจสนามรบ (Battlefield Surveillance):** รายงานสภาพภูมิประเทศ ลมฟ้าอากาศ และความเคลื่อนไหวของข้าศึกแบบ Real-time

## 2. โครงสร้างความสัมพันธ์และการไหลของข้อมูล (Firing Unit Interoperability)
ระบบถูกออกแบบให้ทำงานสอดประสานกันภายในชุดหลักยิง ป. ผ่านวงรอบข้อมูลทางดิจิทัลดังนี้:
- **Observer (ผตน.):** ทำหน้าที่ Input ข้อมูลเป้าหมายผ่านแอปฯ โดยแอปฯ จะทำการตรวจสอบความถูกต้องของ Logic (Validation) ก่อนส่ง "คำขอยิง" ไปยัง ศอย.
- **Fire Direction Center (ศอย.):** ระบบจะส่งชุดข้อมูลดิจิทัลเพื่อใช้ในการประเมินค่าและเลือกวิธีโจมตี พร้อมคำนวณ "หลักฐานยิง" ให้กับส่วนยิง
- **Firing Battery (ส่วนยิง):** รับคำสั่งยิงและตั้งหลักฐานอาวุธเพื่อทำลายเป้าหมายตามที่ ผตน. ร้องขอ

## 3. ข้อมูลพื้นฐานและตัวแปรส่วนกลาง (Core Variables & Technical Specs)
เพื่อให้ระบบมีความแม่นยำสูงสุดตามหลักมูลฐานการยิง ตัวแปรต้องถูกกำหนดดังนี้:
| ตัวแปร | คำอธิบาย | หน่วยวัด / มาตรฐาน |
|---|---|---|
| Grid Coordinates | พิกัดที่ตั้ง (ต้องรองรับมาตรฐาน 8-10 หลัก) | UTM / Grid Reference |
| Direction (Azimuth) | ทิศทาง (ต้องเลือกระหว่าง Grid และ Magnetic Azimuth) | มิลเลียม (Mils) (0 - 6400) |
| Horizontal Distance | ระยะห่างในแนวระนาบจาก ผตน. ถึงเป้าหมาย | เมตร (Meters) |
| Vertical Interval (VI) | ความสูงต่าง (Input เป็น Signed Integer +/-) | เมตร (Meters) |
| OT Line | แนวตรวจการณ์ (Observer-Target Line) | มิลเลียม (Mils) |

## 4. การออกแบบตรรกะ: วิธีพิกัดกริด (Grid Method Logic)
ใช้สำหรับกรณีที่ ผตน. ทราบพิกัดเป้าหมายที่แน่นอนจากแผนที่หรือเครื่องมือหาพิกัดโดยตรง
- **Inputs:** พิกัดเป้าหมาย (8 หรือ 10 หลัก) และระดับความสูง (Altitude)
- **System Logic:**
  - ระบบทำการตรวจสอบค่าความสัมพันธ์ของพิกัดเป้าหมายกับพื้นที่หวงห้าม (No Fire Area)
  - ส่งค่าพิกัดไปยัง ศอย. โดยตรงเพื่อให้ ศอย. คำนวณหาความสัมพันธ์กับที่ตั้งปืน (Battery Location) ตามหลักการยิงจำลอง

## 5. การออกแบบตรรกะ: วิธีโพลาร์ (Polar Plot Method Logic)
ใช้เมื่อ ผตน. ทราบพิกัดตนเองและใช้เครื่องมือวัดทิศทางและระยะไปยังเป้าหมาย
- **Inputs:** พิกัดที่ตั้ง ผตน. (Observer Location), ทิศทางไปยังเป้าหมาย (Direction), ระยะทาง (Distance), และความสูงต่าง (VI)
- **System Logic:**
  - กำหนดจุดเริ่มต้นจากพิกัด ผตน. และสร้าง แนวตรวจการณ์ (OT Line)
  - คำนวณแยกส่วนระหว่างระยะในแนวระนาบ (Horizontal Distance) และส่วนเปลี่ยนความสูง (Vertical Interval)
  - ใช้ตรรกะทางตรีโกณมิติบนฐานวงกลม 6400 มิลเลียม เพื่อคำนวณหาพิกัดเป้าหมายใหม่ (Target Grid) ก่อนส่งให้ ศอย.

## 6. การออกแบบตรรกะ: การย้ายจากจุดที่ทราบที่ตั้ง (Shift from a Known Point Logic)
เป็นตรรกะที่มีความซับซ้อนที่สุด โดยอ้างอิงจากจุดอ้าง (Known Point) ในบัญชีเป้าหมายขั้นต้น
- **Inputs:**
  - การเลือกจุดอ้าง (Known Point Grid)
  - การเลี้ยว (Lateral Shift - ซ้าย/ขวา) เป็นเมตร
  - การเปลี่ยนระยะ (Range Shift - เพิ่ม/ลด) เป็นเมตร
  - การเปลี่ยนความสูง (Vertical Shift - ขึ้น/ลง) เป็นเมตร
- **System Logic:**
  - ระบบต้องคำนวณหา OT Factor (R) โดย R = ระยะจาก ผตน. ถึงจุดอ้าง (หน่วยเป็นพันเมตร)
  - ใช้สูตร W = R x m (W คือระยะเบี่ยงแนวข้าง, m คือมุมเบี่ยงเป็นมิลเลียม) เพื่อตรวจสอบความสัมพันธ์ของแนวทิศ
  - คำนวณแบบเวกเตอร์ (Vector-based) จากจุดอ้างเดิม โดยใช้ค่าเบี่ยงเบนที่ป้อนเข้าเพื่อสร้างพิกัดเป้าหมายใหม่ที่สัมพันธ์กับ แนวตรวจการณ์ (OT Line) เดิม

## 7. ข้อกำหนดด้านความแม่นยำและรายงานผล (Accuracy & Safety Visualization)
ระบบต้องแสดงผลด้านความแม่นยำเพื่อความปลอดภัยของฝ่ายเรา:
- **100% Rectangle (สี่เหลี่ยมผืนผ้า 100%):** UI ต้องแสดงรูปขบวนอาการกระจาย (Dispersion Pattern) รอบจุดปานกลางมณฑล (MPI) โดยคำนึงถึงค่าความคลาดเคลื่อนทางระยะ (PER) และทางทิศ (PED)
- **Checklist การรายงานผล (Surveillance of Fires):**
  - Nature of Burst: [ ] Air (ระเบิดอากาศ) [ ] Impact (ระเบิดกระทบแตก) [ ] Crater (ระเบิดขุดบ่อ)
  - Correction Code: การแก้ทิศทาง (Deflection Correction) และการแก้ระยะ (Range Correction) ในหน่วย 10-50 เมตร
  - Effect on Target: [ ] ทำลาย [ ] ข่ม [ ] รบกวน

## 8. ข้อจำกัดและข้อควรระวังทางเทคนิค (Technical Constraints)
**Design Considerations for Battlefield Environment:**
- **Tactical Stealth:** แอปฯ ต้องรองรับการทำงานร่วมกับ เครื่องควบคุมระยะไกล (Remote Control) เพื่อให้ ผตน. สามารถสั่งการได้จากระยะที่ปลอดภัยจากการถูกตรวจพบ
- **Offline-First & Data Integrity:** ระบบต้องคำนวณผลได้โดยไม่ต้องพึ่งพาสัญญาณเครือข่ายตลอดเวลา และใช้ระบบ Burst Transmission ในการส่งคำขอยิงเพื่อลดโอกาสที่ศัตรูจะตรวจจับสัญญาณวิทยุได้
- **Operational Speed:** UI ต้องลดขั้นตอนการพิมพ์ (Minimum Keystrokes) เพื่อรองรับการยิงอย่างจู่โจม (Surprise) และเพิ่มอำนาจการทำลาย
- **Light & Camouflage:** การออกแบบหน้าจอต้องมีโหมดความสว่างต่ำ (Low-Light Mode) เพื่อป้องกันการสะท้อนของแสงที่อาจเปิดเผยที่ตั้งตรวจการณ์
